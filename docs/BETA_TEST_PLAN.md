## 🧪 測試場景

### 場景 1: 門診病歷記錄

**流程:**
1. 醫師看診時使用語音輸入
2. 系統即時轉錄並生成 SOAP
3. 醫師檢視並調整 SOAP 內容
4. 儲存病歷

**評估指標:**
- 語音辨識準確率 (≥90%)
- SOAP 生成準確率 (≥85%)
- 平均就診時間變化

### 場景 2: 急診快速記錄

**流程:**
1. 急診醫師使用語音快速記錄
2. 系統即時生成初步病歷
3. 醫師補充關鍵資訊
4. 快速完成病歷

**評估指標:**
- 記錄時間節省 (≥50%)
- 關鍵資訊完整性 (≥95%)
- 系統回應時間 (<3 秒)

### 場景 3: 複雜病例記錄

**流程:**
1. 多症狀、多系統病例
2. 系統整合多項資訊
3. 生成完整 SOAP 病歷
4. 醫師審核並修正

**評估指標:**
- 複雜病例處理能力
- 多症狀整合準確性
- 醫師滿意度

## 🔧 技術需求

### 硬體需求

- 電腦：Windows/Mac/Linux
- 記憶體：8GB RAM 以上
- 網路：內網連線 (192.168.x.x)
- 麥克風：USB 麥克風或耳機麥克風

### 軟體需求

- 瀏覽器：Chrome 90+ 或 Firefox 88+
- 作業系統：Windows 10+, macOS 11+, Ubuntu 20.04+

### 網路需求

- API 伺服器：http://192.168.x.x:8000
- 延遲：<100ms
- 頻寬：≥1 Mbps
