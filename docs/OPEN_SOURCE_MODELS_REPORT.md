## 📊 執行摘要

本報告評估了適用於醫療語音轉文字（ASR）後處理與 SOAP 病歷生成的開源 LLM 模型，重點關注：

| 條件 | 要求 |
|------|------|
| 參數數量 | < 13B（本地可運行） |
| 中文支援 | 繁體/簡體中文優先 |
| 醫療領域 | 有醫療微調版本佳 |
| 部署方式 | Ollama / llama.cpp 本地部署 |

### NotebookLM 研究發現（2026-03-21）

根據 NotebookLM 對 22 個來源的分析，開源醫療 LLM 的關鍵發現：

1. **哈佛研究證實：** Llama 3.1 405B 在醫學診斷複雜病例的判斷上已超越 GPT-4（70% vs 64% 準確率）
2. **台灣醫療趨勢：** 75% 醫療單位最期待「醫療紀錄摘要」應用，27% 採用在地部署
3. **最佳實踐：** 需整合 RAG、知識圖譜注入、臨床指南約束以減少幻覺

### SOAP 病歷生成最佳實踐（NotebookLM 研究）

根據 2025 年哈佛醫學院研究與台灣醫療產業調查，SOAP 病歷生成的關鍵實踐：

#### 1. 減少模型幻覺
- **檢索增強生成 (RAG)**：引入外部醫學知識庫
- **知識圖譜注入**：整合 ICD-10、SNOMED CT 等標準術語
- **循證醫學約束**：對照臨床指南進行驗證

#### 2. 品質保證機制
- **不確定性量化**：當置信度低於閾值時自動觸發人工複核
- **CHAI 責任型 AI 指引**：建立臨床 QA/QC 架構
- **模型自我評估**：使用 PiCO 方法校準輸出

#### 3. 工作流程整合
- **UI 贴近臨床習慣**：減少學習曲線
- **嵌入現有 HIS 系統**：透過 API 整合
- **標準化資料格式**：解決 60% 單位面臨的資料格式不一致問題
