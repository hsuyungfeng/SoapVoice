## 🚀 快速開始

### 1. 啟動 API 服務

```bash
cd /home/hsu/Desktop/SoapVoice
uv run uvicorn src.main:app --host 0.0.0.0 --port 8000 &
```

### 2. 驗證服務

```bash
curl http://localhost:8000/health
# 預期輸出：{"status":"healthy","service":"SoapVoice API","version":"0.1.0"}
```

### 3. 執行測試

```bash
uv run python scripts/test_recording.py
```

**選擇測試模式:**
- `1` - WebSocket 連接測試
- `2` - 音頻串流測試（需要麥克風）
- `3` - 文字輸入測試（模擬）← **推薦**
- `4` - 完整測試

## 🎤 音頻串流測試（選擇 2 或 4）

### 測試流程

```
1. 連接 WebSocket ✓
2. 等待連接確認
3. 發送 "start" 訊息
4. 等待 "stream_started" 回應
   ⏱️ Whisper 模型載入中（可能需要 10-20 秒）
5. 開始錄音（10 秒）
6. 發送 150 個音頻塊
7. 發送 "end" 訊息
8. 等待最終轉錄結果
```

## 🔧 常見問題

### 問題 1: 「串流會話未開始」錯誤

**原因:** Whisper 模型未正確載入

**解決方案:**
1. 確保 API 服務已重啟
2. 等待足夠時間讓模型載入（10-20 秒）
3. 查看 API 日誌確認模型載入狀態

### 問題 2: ALSA 警告

```
ALSA lib pcm_dsnoop.c:567:(snd_pcm_dsnoop_open) unable to open slave
```

**說明:** 這是 Linux 系統常見的音頻警告，**不影響功能**。

### 問題 3: 等待回應超時

**原因:** 
- Whisper 模型首次載入需要時間
- 網絡連接問題

**解決方案:**
- 等待更長時間（最多 60 秒）
- 檢查 API 服務是否運行
- 查看 API 日誌

## 📝 查看日誌

### API 日誌

```bash
# 查看最近的日誌
journalctl -n 100 --no-pager | grep -E "(Whisper|websocket|start_stream)"

# 或查看特定日誌文件
tail -f /tmp/api.log
```

### 測試日誌

測試腳本會顯示詳細的進度資訊：
- WebSocket 連接狀態
- 模型載入狀態
- 音頻塊發送進度
- 轉錄結果
